# Spec 추천 목록

PRD를 기반으로 중복 없이 핵심 기능만 추출한 5가지 Spec입니다.
개발 우선순위 순으로 정렬되어 있습니다.

---

## 1. 게임 엔진 코어 (Game Engine Core)

**설명**: 60fps 게임 루프, Canvas 렌더링, 입력 처리를 담당하는 핵심 엔진

**포함 기능**:
- requestAnimationFrame 기반 게임 루프
- Canvas 2D 렌더링 시스템
- 마우스/터치 입력 통합 처리
- 오브젝트 풀링 시스템
- 게임 상태 관리 (메뉴, 플레이, 일시정지, 게임오버)

**명령어**:
```
/speckit.specify 60fps 게임 루프와 Canvas 렌더링을 담당하는 게임 엔진 코어 시스템. requestAnimationFrame 기반 메인 루프, Canvas 2D 렌더링 매니저, 마우스와 터치를 통합 처리하는 입력 시스템, GC 최소화를 위한 오브젝트 풀링, 게임 상태 머신(메뉴/플레이/일시정지/게임오버) 포함. HTML5 Canvas + Vanilla JavaScript 사용.
```

## 2. 비행기 시스템 (Aircraft System)

**설명**: 비행기 스폰, 이동, 경로 추적을 담당하는 시스템

**포함 기능**:
- 비행기 엔티티 (소형기, 대형기, 헬기)
- 화면 가장자리 랜덤 스폰
- 경로 따라 이동하는 AI
- 속도/크기별 차별화된 특성
- 착륙 판정 로직

**명령어**:
```
/speckit.specify 비행기 스폰과 이동을 관리하는 항공기 시스템. 비행기 종류별 클래스(소형기-빠름, 대형기-느림, 헬기-중간), 화면 가장자리에서 랜덤 스폰하는 스포너, 주어진 경로를 따라 부드럽게 이동하는 경로 추적 AI, 활주로 착륙 판정 로직 포함. 오브젝트 풀링과 호환되도록 설계.
```

## 3. 경로 드로잉 시스템 (Path Drawing System)

**설명**: 플레이어가 비행기 경로를 그리는 핵심 인터랙션 시스템

**포함 기능**:
- 비행기 선택 (탭/클릭)
- 드래그로 경로 그리기
- 경로 시각화 (점선/곡선)
- 경로 스무딩 처리
- 활주로까지 경로 연결

**명령어**:
```
/speckit.specify 플레이어가 비행기 경로를 그리는 경로 드로잉 시스템. 비행기 탭/클릭 선택, 드래그로 경로 포인트 수집, 베지어 또는 스플라인 기반 경로 스무딩, 점선으로 경로 시각화, 활주로까지 경로 자동 연결 기능 포함. 마우스와 터치 입력 모두 지원.
```

## 4. 충돌 및 점수 시스템 (Collision & Score System)

**설명**: 비행기 간 충돌 감지와 점수 계산 시스템

**포함 기능**:
- 비행기 간 충돌 감지
- 공간 분할 최적화 (옵션)
- 점수 계산 (비행기 종류별)
- 콤보 시스템 (연속 착륙 보너스)
- 게임오버 트리거

**명령어**:
```
/speckit.specify 비행기 충돌 감지와 점수 계산 시스템. 원형 충돌 감지로 비행기 간 충돌 판정, 성능 최적화를 위한 거리 기반 사전 필터링, 비행기 종류별 차등 점수, 연속 착륙 콤보 보너스 계산, 충돌 시 게임오버 트리거 포함. LocalStorage에 최고 점수 저장.
```

## 5. 공항 레벨 시스템 (Airport Level System)

**설명**: 다양한 공항 맵과 레벨 관리 시스템

**포함 기능**:
- 공항 맵 데이터 구조
- 활주로 배치 (2~4개)
- 레벨별 난이도 설정 (스폰 빈도, 비행기 속도)
- 시간/날씨 비주얼 효과
- 레벨 언락 시스템

**명령어**:
```
/speckit.specify 다양한 공항 맵과 레벨을 관리하는 시스템. 공항 맵 JSON 데이터 구조(활주로 위치, 스폰 포인트), 레벨별 2~4개 활주로 배치, 난이도 설정(스폰 간격, 비행기 속도 범위), 낮/밤 및 날씨 시각 효과, LocalStorage 기반 레벨 언락 저장 포함. 최소 5개 공항 레벨 지원.
```

## 개발 순서 권장

```
1. 게임 엔진 코어     → 기반 시스템
2. 비행기 시스템      → 핵심 엔티티
3. 경로 드로잉 시스템  → 핵심 인터랙션
4. 충돌 및 점수 시스템 → 게임 규칙
5. 공항 레벨 시스템   → 콘텐츠 확장
```

각 Spec은 독립적으로 개발 가능하며, 위 순서대로 진행하면 점진적으로 플레이 가능한 게임이 완성됩니다.
